Your output MUST be a valid JSON object, containing ONLY the JSON. Do not include any additional text, comments, or explanations.

Extract the following information into a JSON object with these keys:

* "copyrights": A JSON array of strings. Each string must be an EXACT, verbatim copy of a copyright statement found in the text.
    * Identify copyright statements by the presence of "copyright", "(C)", or "(c)".
    * "All rights reserved." remarks are part of the copyright statement.
    * Do not include license information.
    * Preserve original whitespace.
    * Escape newlines as `\n` and tabs as `\t`.
    * Remove leading/trailing comment delimiters (e.g., `/*`, `*/`, `//`, `*`) if they are part of a comment block and not the copyright text itself.
    * Treat blocks of multiple copyright statements with no intervening lines as a single copyright string.

* "holders": A JSON array of strings. Each string must be the name of a copyright holder.
    * Extract names directly from the identified copyright statements.
    * Exclude dates or other non-holder information.
    * Include the holder's mail address if present in the copyright statement.

* "authors": A JSON array of strings. Each string must be the name of an author mentioned outside copyright statements.
    * Authorship implies anyone who contributed to the material (e.g., editors, contributors). Maintainers are not considered authors.
    * Include the author's mail address if provided.

**Mail Address Normalization:** For both "holders" and "authors" mail addresses, normalize them to lowercase and use delimiters "<" and ">" it not present.

Ignore all content not relevant to copyright or authorship information (e.g., source code, license text).
